# Support

Thanks for using the iXBRL skill. Here is how to get help.

## Questions about using the skill

- **Read the docs first.** [`README.md`](README.md) covers install and
  compatibility; [`SKILL.md`](SKILL.md) is the agent entrypoint; the
  [`references/`](references/) directory has primary-source-cited notes on the
  spec, taxonomies, ESEF, EDGAR/EFM, dimensions, validation, and more.
- **Ask a question.** Open a [GitHub Discussion](https://github.com/MaxSchoon/ixbrl/discussions)
  if Discussions are enabled, or email **contact@doc2ixbrl.com** for general
  questions and collaboration proposals.

## Found a bug, a wrong rule, or a misfiring trigger?

Open an [issue](https://github.com/MaxSchoon/ixbrl/issues/new/choose) and pick
the template that fits — spec-citation correction, regulator update, scaffold
regression, trigger misfire, or enhancement. The templates ask for the
primary-source citation that every factual change needs (see
[`CONTRIBUTING.md`](CONTRIBUTING.md)).

## Security or filing-integrity concerns

**Do not open a public issue.** A scaffold or rule that would produce an
apparently-valid but non-compliant filing is reported privately — see
[`SECURITY.md`](SECURITY.md).

## What support is and isn't

This is a community, best-effort, open-source project maintained on a
volunteer basis. It is **not** legal, accounting, audit, or filing advice, and
there is no SLA. For a regulated filing, always verify against the live
publisher source and involve a qualified professional.
